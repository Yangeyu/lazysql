# ADR 0013：SQL 编辑器两档展示——默认一行回显条，显式展开编辑（修订 0010 代价 3）

- 状态：已接受
- 日期：2026-07-15
- 修订：2026-07-16（收起态鼠标选择不触发展开）；2026-07-21（展开态点击聚焦）
- 关联：ADR 0010（多行 SQL 编辑器）——**修订其「代价 3：编辑面板定高 10 行」**，其余 0010 决策（textarea 自持 + store 镜像、键位）不变；ADR 0006（lazygit 导航）、ADR 0007（keymap 分发）。

## 背景

网格是主视图：用户绝大部分时间在浏览数据，SQL 草稿是空的。此时常驻的 10 行编辑面板里，唯一有信息量的是 `statement` 回显（以 placeholder 暗色显示）——为一行信息付 10 行屏幕。

设计过程中依次否决了两个替代：

1. **auto-grow（高度随内容行数派生）**：省一个键位，但每加删一行网格就跳一次，且展开画布是"长"出来的而非立即给足。
2. **展开态从 `focus === 'editor'` 派生**：零新增状态，但「改 SQL → 运行 → 看网格 → 再改」的迭代循环里焦点每次离开编辑器面板就收起，布局抖动比省下的空间更烦人。

最初把任一档位的「点击面板」都作为展开并聚焦的入口。实际使用发现，收起回显条开始拖选时，`onMouseDown`
会立刻展开面板，后续拖拽坐标落到重排后的 ask/textarea 行，复制结果因此错位；因此 2026-07-16 曾将整个 SQL
面板改成只选择、不聚焦。后续使用表明这又把两个档位混为一谈：展开态已经是可编辑 surface，点击却不能输入，
不符合鼠标交互预期。展开态聚焦不会改变档位或布局，不存在收起态的坐标漂移问题，故只保留收起态的隔离。

## 决策

编辑器有**两档**，档位是**粘性 UI 偏好**（`editorExpanded: boolean`，同 `sidebarWidth` 一个范畴），与焦点、内容、查询运行全部解耦：

- **收起（默认）**：3 行带边框回显条——`SQL>` + 网格背后语句的单行截断（browse SQL 或刚执行的查询），只读、**不可聚焦**、无 ask 行；存在未完成草稿时尾部标 `(draft)`（草稿始终留在 store 镜像里，不丢）。网格比 0010 的定高多得 7 行。
- **展开**：完整编辑面板（ask 行 + 多行 textarea + 补全/反馈），高度维持 0010 的 10 行（曾试过 `⌊rows/2⌋` 封顶 15，展开时挤占网格过多，退回）。**粘性**：运行查询、esc 回网格、Tab 切面板都不改变档位——迭代循环零布局跳动。

**转换与输入边界**：

| 触发 | 效果 |
|---|---|
| `:` | 展开 + 聚焦（`focusPane('editor')` 蕴含展开——回显条里无法编辑） |
| 全局 `^G` | 展开 + 聚焦 NL ask 输入 |
| `^O`（全局与编辑器内均可） | 切换档位；收起时若焦点在编辑器则移到 grid，开着的 ask 行取消 |
| 鼠标点击 / 拖拽收起回显条 | 只做文本选择与复制；不展开、不改变 pane focus |
| 鼠标按下展开面板 | 聚焦 SQL editor；拖拽仍选择并复制，不改变档位 |
| esc（编辑器内） | 焦点回 grid，**面板保持展开** |

## 代价 / 边界

1. 花掉一个键位 `^O`（全局 + 编辑器上下文各注册一条，指向同一 action）。
2. 收起态的多行草稿不可见（只有 `(draft)` 徽标）——按 `:` 一步找回；接受，避免回显条膨胀。
3. `^G` 从编辑器专属提升为全局键（收起态一步进入问答）；非 kitty 终端的退化行为与 0010 相同。
4. 展开态拖选也会把 pane focus 交给 editor；这是与编辑 surface 交互的正常语义，且档位不变，所以选择坐标稳定。

**实现约束（踩过的坑）**：两档共用**一个常驻 `<textarea>`**，收起 = `visible={false}`（yoga display:none），
**不是卸载**。卸载重挂会让 widget 以自己的 boot 光标（focus 置于文本末尾）重生，并抢在 reconcile effect
之前经 `onCursorChange` 回显、污染 store 的 `editorCaret` 镜像——0010 的镜像协议只为常驻 widget 设计。
回归测试：`editorGear.test.tsx` 的 collapse-expand caret 用例。

## 结论

「回显」与「编辑」是两个职责，分属两档 surface；档位归用户手动控制（粘性），进入编辑的键盘路径自动带展开。
收起态鼠标只选择/复制，展开态鼠标聚焦且仍可选择/复制；两者都不切换档位。0010 的定高取舍在"网格是主视图"
的使用事实下修订为两档定高——每档内仍不随内容增长，布局可预测性保持。
